# 프로젝트 아이디어 & 의사결정 기록

> 프로젝트 방향에 대한 고민, 후보 아이디어, 결정 사항을 시간순으로 기록합니다.
> 결정이 바뀌면 지우지 말고 아래에 새 항목을 추가합니다. (사고 과정 자체가 포트폴리오)

## 배경 (2026-07)

- 팀: 3년차 태블로 엔지니어 2인. SQL/Python 기초, BI 능숙.
- 목표: 데이터 분석가 직무 이직. BI 대시보드가 아니라 **스스로 기획 → KPI 설계 → A/B 테스트 → 검증**하는 분석 경험을 만드는 것.
- 부가 목표: GitHub 브랜치/PR 기반 협업 워크플로 습득.
- 관심사: 화장품(올리브영), 감성 숙소, 맛집(캐치테이블), 옷/인스타.

## 후보 아이디어

### 1순위 — 감성 숙소 큐레이팅 사이트 실데이터 분석 ✅ (현재 진행 방향)

- 지인이 운영하는 Airbnb류 감성 숙소/펜션 큐레이팅 웹사이트의 실데이터를 받을 가능성 있음.
- **선정 이유**: 공개 데이터가 아닌 실서비스 데이터 + 실제 운영자와의 커뮤니케이션이 들어가는 프로젝트는 그 자체로 차별화됨. 분석 결과가 실제 서비스에 반영되면 최상급 스토리.
- 진행 흐름:
  1. 운영자 인터뷰 — 수익 구조 파악 (예약 수수료? 광고?)
  2. KPI 트리 설계 — 예: 매출 = 방문자 × 상세 클릭률 × 예약 전환율 × 객단가
  3. 퍼널/코호트 분석 — 유입 채널별 전환율, 재방문율, 노출 순서와 클릭 관계
  4. A/B 테스트 — 예: 메인 큐레이션 정렬 방식(인기순 vs 신규순)에 따른 클릭률 차이.
     트래픽이 적으면 표본 크기 계산 → 검정력 분석 → 시뮬레이션 검증으로 설계서 완성.
  5. 운영자에게 실제 제안 → 반영 여부와 개선 효과까지 추적
- ⚠️ 주의: 데이터 받기 전에 **비식별화 범위와 공개 가능 범위를 운영자와 서면 합의**할 것.
  원본 데이터는 레포에 절대 올리지 않음 (.gitignore 처리 완료).

### 2순위 — Inside Airbnb 서울 데이터 (백업 플랜)

- 실데이터를 못 받게 될 경우 대비. 공개 데이터지만 감성 숙소 관심사와 부합.
- 예시 기획: 가격 대비 리뷰 점수로 저평가 숙소 발굴 → 호스트 대상 가격 전략 제안.

### 3순위 — 올리브영 리뷰 분석 (보류)

- 보류 이유: 크롤링 필요(약관 이슈), 행동 데이터가 아닌 텍스트 데이터라
  A/B 테스트·KPI 설계 경험을 쌓기 어려움. 목표와 결이 다름.
